Lecture 26: Nonblocking I/O

Mon, Mar 9th Slides: [live] [markdown] [pdf]

I want to spend some time today and Wednesday (and possibly Friday) talking about nonblocking I/0, which can be used to minimize the number of blocking system calls (e.g. slow system calls like accept and read are transformed into ones that return immediately, even when no data is available). By doing so, we'll be able to implement robust servers that can manage tens of thousands of open connections and serving them all in a single thread of execution!

Note that this material is optional, in the sense that it doesn't contribute to any assignment solution, and it won't be tested on the final exam. It is, however, awesome material that many system engineers working out in industry told me was a programming model they use more often than multithreading and/or multiprocessing when it comes to managing I/O.
